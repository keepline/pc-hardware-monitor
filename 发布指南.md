# 🚀 发布指南 - 让小伙伴使用你的工具

## 📋 完整步骤

### 第一步：打包应用

```bash
# 确保已经安装了依赖
npm install

# 打包Windows可执行文件
npm run build
```

等待打包完成，会在 `dist` 目录生成以下文件：
- `PC硬件监控工具-1.0.0-portable.exe` - 便携版（推荐）
- `PC硬件监控工具-Setup-1.0.0.exe` - 安装版

### 第二步：提交代码到GitHub

```bash
# 添加所有更改
git add .

# 提交更改
git commit -m "更新：添加打包配置和完整说明"

# 推送到远程仓库
git push origin master
```

### 第三步：创建GitHub Release

1. **访问你的GitHub仓库**
   ```
   https://github.com/keepline/pc-hardware-monitor
   ```

2. **点击右侧的 "Releases"**

3. **点击 "Create a new release"**

4. **填写发布信息**：
   - Tag version: `v1.0.0`
   - Release title: `PC硬件监控工具 v1.0.0`
   - Description（示例）:
     ```markdown
     ## ✨ 新版本发布！
     
     ### 功能特性
     - ✅ 实时监控CPU、内存、GPU、磁盘、网络等硬件信息
     - ✅ 黑色科技风格界面
     - ✅ 自动刷新数据（每5秒）
     - ✅ 智能导航标签
     
     ### 下载方式
     - **便携版**：下载 `PC硬件监控工具-1.0.0-portable.exe` 直接运行
     - **安装版**：下载 `PC硬件监控工具-Setup-1.0.0.exe` 完整安装
     
     ### 系统要求
     - Windows 10/11（64位）
     
     ---
     
     **首次发布，欢迎反馈问题！**
     ```

5. **上传文件**：
   - 将 `dist` 目录下的两个 `.exe` 文件拖拽到上传区域
   - 等待上传完成

6. **点击 "Publish release"**

### 第四步：分享给小伙伴

发布完成后，将以下链接发给小伙伴：

#### 🔗 方式一：仓库主页（推荐）
```
https://github.com/keepline/pc-hardware-monitor
```
小伙伴可以：
- 查看完整的README说明
- 点击右侧的 "Releases" 下载最新版本
- 查看源代码

#### 🔗 方式二：直接下载页面
```
https://github.com/keepline/pc-hardware-monitor/releases
```
小伙伴可以直接看到所有版本和下载链接

#### 🔗 方式三：直接下载链接
```
https://github.com/keepline/pc-hardware-monitor/releases/download/v1.0.0/PC硬件监控工具-1.0.0-portable.exe
```
点击直接下载便携版

## 📝 小伙伴使用步骤

告诉你的小伙伴，他们只需要：

### ✅ 使用便携版（最简单）
1. 访问：https://github.com/keepline/pc-hardware-monitor/releases
2. 下载：`PC硬件监控工具-1.0.0-portable.exe`
3. 双击运行即可，**无需安装！**

### ✅ 使用安装版
1. 访问：https://github.com/keepline/pc-hardware-monitor/releases
2. 下载：`PC硬件监控工具-Setup-1.0.0.exe`
3. 运行安装程序，按提示安装
4. 从桌面或开始菜单启动

### ✅ 从源码运行（开发者）
1. 克隆仓库：`git clone https://github.com/keepline/pc-hardware-monitor.git`
2. 安装依赖：`npm install`
3. 运行：`npm start`

## ⚠️ 注意事项

1. **首次打包时间较长**，需要下载Electron二进制文件
2. **文件较大**（约150MB），因为包含了完整的Electron运行环境
3. **Windows可能提示"未知发布者"**，这是正常的，点击"仍要运行"即可
4. **建议同时提供便携版和安装版**，让用户自由选择
